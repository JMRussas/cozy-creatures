# Asset Pipeline

## Overview

Assets come from 158 purchased Unity Asset Store packages stored in the UnityAssets project at `~/Git/UnityAssets/`. The social catalog tool classifies these for use in the social app.

## Social Catalog

### Building the Catalog

```bash
python tools/build_social_catalog.py              # Build/rebuild
python tools/build_social_catalog.py summary       # Print stats
python tools/build_social_catalog.py recommend     # Recommendations by system
```

### How Classification Works

Each package is scored against 15 categories using three signals:
1. **category_dir** — Unity Store category from download path (+10 points)
2. **name** — Package name keyword matching (+5 points)
3. **files[]** — File path keyword sampling (+2 points)

Highest score wins. Packages are also rated for social app relevance (high/medium/low/none) and tagged with specific use cases (avatar_base, emote_animations, etc.).

### Categories

| Category | What It Covers |
|----------|---------------|
| characters_avatars | Character models, humanoid rigs, creatures |
| animations | Animation clips, controllers, motion packs |
| environments | Rooms, dungeons, outdoor scenes, themed spaces |
| ui_gui | GUI frameworks, icon packs, UI samples |
| networking | Multiplayer, sync, lobby/room systems |
| ai_behavior | Behavior trees, pathfinding, dialogue, state machines |
| vfx_particles | Particle effects, magic VFX, spell effects |
| audio | Sound effects, music, footsteps |
| weapons_combat | Weapons, combat assets |
| vehicles | Cars, tanks, ships |
| tools_systems | Character controllers, inventory, save systems |
| prototyping | Starter packs, prototype assets, samples |
| terrain_nature | Nature, skyboxes, weather, terrain tools |
| props | Decorative objects, furniture, barrels |
| textures_materials | Ground textures, materials, PBR sets |

### Relevance Tiers

- **high** — Directly useful (avatars, UI, networking, core animations)
- **medium** — Supporting (environments, AI, systems, props)
- **low** — Niche (VFX, audio, terrain, textures)
- **none** — Not applicable (weapons, vehicles, combat)

## Importing Assets into the Project

1. Run `python tools/build_social_catalog.py recommend` to see what's available
2. Find the package in `~/Git/UnityAssets/extracted/<PackageName>/`
3. Copy the relevant folders into `Assets/ThirdParty/<PackageName>/`
4. In Unity Editor: right-click `ThirdParty/` > Reimport All

Or import directly from .unitypackage:
1. Find package in `%APPDATA%/Unity/Asset Store-5.x/<Publisher>/<Category>/`
2. In Unity: Assets > Import Package > Custom Package

## Source Data

The master catalog lives at `~/Git/UnityAssets/catalog.json` (158 packages, ~85K lines). It's generated by `~/Git/UnityAssets/tools/catalog_assets.py scan` and contains per-package metadata: file lists, file type distributions, content flags, and Unity Store metadata.

## Gotchas

- catalog.json is UTF-8 encoded — always open with `encoding='utf-8'`
- Some packages have `"error"` keys if they couldn't be inspected
- Package names come from the .unitypackage filename, not internal metadata
- The `files[]` array can be very large for big packages — the classifier only samples the first 30-50 entries
